# AI Usage Policy

## Overview

AI-assisted tools are permitted in xLaDe. Contributors may use AI for coding, documentation, refactoring, analysis, and related tasks.

## Contributor Responsibility

Contributors remain fully responsible for all submitted work, regardless of AI assistance.
Final authorship and accountability belong to the contributor submitting the change.

## Disclosure

Disclosure of AI use is optional.
Voluntary disclosure of tools or models used is appreciated for transparency and reproducibility.

## Understanding Requirement

Contributors should understand the code or documentation they submit and be able to explain its behavior and purpose if requested.
Blind or unreviewed use of AI-generated output is discouraged.

## Research Integrity

AI-generated output must not replace critical reasoning, verification, or human review, especially in research-oriented parts of the project.

## Prohibited Practices

The following are prohibited:

- Submitting AI-generated code without understanding behavior and implications.
- Using AI output without review, validation, and basic testing.
- Using AI to alter or translate core/trusted modules without full understanding and review.
- Relying on low-quality model output when it materially harms correctness, safety, or maintainability.

## Recommended Practices

Contributors are encouraged to:

- Review and edit AI output before submission.
- Run relevant tests and checks for AI-assisted changes.
- Keep AI-assisted changes small and easy to audit.
- Include rationale in commit messages and PR descriptions for non-trivial AI-assisted edits.

## Policy Updates

This policy may be updated over time based on project needs, contributor discussion, and changes in AI tooling.
